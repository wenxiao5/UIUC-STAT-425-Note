Unbalanced ANOVA

As you can observe, the outputs we get are different in cases (i) and (ii). The reason is that R is performing a sequential calculation of the Sums of Squares and a conditional testing of the models. So, let’s investigate what are the hypothesis tests that are being done behind the scenes:

(i) treat first, poison second

For the first line that corresponds to treat R is testing the following hypothesis: \[\begin{cases} &H_0: y_{ijk} = \mu + \varepsilon_{ijk}\\ &H_{\alpha}: y_{ijk} = \mu + \alpha_i + \varepsilon_{ijk}\\ \end{cases} \] where \(\alpha_i\) corresponds to the treat factor. The \(F\) value for this test is 29.6807 and the p-value 9.986e-10.

For the second line that corresponds to poison R is testing the following hypothesis: \[\begin{cases} &H_0: y_{ijk} = \mu + \alpha_i + \varepsilon_{ijk}\\ &H_{\alpha}: y_{ijk} = \mu + \alpha_i + \beta_j + \varepsilon_{ijk}\\ \end{cases} \] where \(\beta_j\) corresponds to the poison factor. The \(F\) value for this test is 77.6094 and the p-value 1.362e-13. So, as you can see the model under the \(H_0\) is the model with the treat factor, not the null model.

For the third line that corresponds to treat:poison R is testing the following hypothesis: \[\begin{cases} &H_0: y_{ijk} = \mu + \alpha_i ++ \beta_j+ \varepsilon_{ijk}\\ &H_{\alpha}: y_{ijk} = \mu + \alpha_i + \beta_j + (\alpha\beta)_{ij} \varepsilon_{ijk}\\ \end{cases} \] where \((\alpha\beta)_{ij}\) corresponds to the treat*poison factor. The \(F\) value for this test is 1.4592 and the p-value 0.2207. So, as you can see the model under the \(H_0\) is the model with the treat and poison factors.

So, if we change the order in the variables in the model, then R is going to perform a different series of sequential tests.

(ii) poison first, treat second

For the first line that corresponds to poison R is testing the following hypothesis: \[\begin{cases} &H_0: y_{ijk} = \mu + \varepsilon_{ijk}\\ &H_{\alpha}: y_{ijk} = \mu + \beta_j + \varepsilon_{ijk}\\ \end{cases} \] where \(\beta_j\) corresponds to the poison factor. The \(F\) value for this test is 81.0799 and the p-value 7.276-14.

For the second line that corresponds to treat R is testing the following hypothesis: \[\begin{cases} &H_0: y_{ijk} = \mu + \beta_j + \varepsilon_{ijk}\\ &H_{\alpha}: y_{ijk} = \mu + \alpha_i + \beta_j + \varepsilon_{ijk}\\ \end{cases} \] where \(\alpha_i\) corresponds to the treat factor. The \(F\) value for this test is 27.3670 and the p-value 2.706-09. So, as you can see the model under the \(H_0\) is the model with the poison factor, not the null model.

For the third line that corresponds to treat:poison R is testing the following hypothesis: \[\begin{cases} &H_0: y_{ijk} = \mu + \alpha_i ++ \beta_j+ \varepsilon_{ijk}\\ &H_{\alpha}: y_{ijk} = \mu + \alpha_i + \beta_j + (\alpha\beta)_{ij} \varepsilon_{ijk}\\ \end{cases} \] where \((\alpha\beta)_{ij}\) corresponds to the treat*poison factor. The \(F\) value for this test is 1.4592 and the p-value 0.2207. So, as you can see the model under the \(H_0\) is the model with the treat and poison factors, and that is why this line is the same for both cases.

Partial \(F\) tests in R

So, in order to avoid confusion with the order we input variables in the model, it is preferred to do a partial \(F\) test. In this way, we ensure that each time we understand which is the model under the \(H_0\) and which under the alternative. So, we start by fitting the full model with interactions and the additive model to test for interactions:

ginv1 <- lm(time^-1 ~poison*treat, data=newrats)
ginv2 <- lm(time^-1 ~poison+treat, data=newrats)
anova(ginv2, ginv1)

ANOVA with \(n=1\)

In this example, we are going to investigate what happens when we only have one observation per factor level combination.

Consider the composite data set from the faraway library: The composite data frame has 9 rows and 3 columns. The data comes from an experiment to test the strength of a thermoplastic composite depending on the power of a laser and speed of a tape.

library(faraway)
library(ggplot2)
## Warning: package 'ggplot2' was built under R version 3.6.2

attach(composite)
names(composite)
## [1] "strength" "laser"    "tape"

In this data set, we only have 9 data points. If we want to fit the full interaction model \[y_{ijk} = \mu + \alpha_i + \beta_j + (\alpha\beta)_{ij}+ \varepsilon_{ijk}\] where \(\alpha_i\) corresponds to the laser strength with \(i=1,2,3\) and \(\beta_j\) corresponds to the speed of tape with \(j=1,2,3\), then we need to estimate 2 parameters corresponding to factor A, 2 parameters corresponding to factor B, 4 parameters corresponding to interaction AB, plus the baseline \(\mu\), which means 9 parameters in total! This means that there are no DFs left to estimate the error, which leads to the following output in R:
